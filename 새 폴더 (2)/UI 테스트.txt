1. 페이퍼 프로토타입 (Paper Prototype) 평가
	프로토타입의 가장 빠른 방법으로 제품의 전반적인 컨셉과 흐름을 잘 보여주며, 
	보는 사람들이 최종 제품에 대한 기대를 갖지 않고 
	더 자유롭게 의견을 개진하면서 발전시킬 수 있는 방법	

2. 인터랙션 디자인
	입출력 장치를 매개로 디지털 시스템과 사람이 주고 받는 일련의 의사소통 과정,
	사람의 행동과 이에 반응하는 시스템의 절차를 설계하는 것

3. Jakob Nielsen의 사용
	Jakob Nielsen은 사용성을 
	학습용이성, 사용효율성, 기억용이성, 최소한의 에러, 주관적 기쁨의 요견을 
	충족시킴으로써 얻어지는 시스템의 특성이라고 주장

4. 파일럿 테스트
	컴퓨터 프로그램 등의 최신 기술을 개발하여, 
	실제 상황에서 실현하기 전에 소규모로 시험 작동 해보는 것을 의미
	or 대규모 프로젝트를 실행하거나 플랜트를 본격적으로 가동하기 전에, 
	발생할 수 있는 변수를 파악하고 수정하기 위해 모의로 시행해 보는 것

5. 태스크 성공 매트릭스
	태스크가 포함되어 있는 사용성 테스트 연구에서 일반적이고 보편적인 방법,
	개선 이전의 UI/UX 디자인과 개선 이후의 UI/UX 디자인의 태스크 
	성공 여부(이진성공율)의 빈도 분포를 비교하는 것


6. NEM 기법
	대상 제품을 처음으로 사용해보는 초심자와 설계, 개발에 참여한 숙련자가 
	함께 사용성 테스트에 참여했을 때 일 처리 시간을 비교하여
	문제점을 객관적으로 제시하는 사용성 테스트 기법

7. UI 디자인 (User Interface Design)
	기기나 SW의 화면처럼 사람과 접하는 면(인터페이스)를 설계하고 디자인하는 것

8. UI 개선 방안 수립
	수행 순서
	1) 사용성 테스트를 통해 도출된 이슈에 대해 이해
	2) UI 개선 방안 수립 목적을 정의하고, 참여 인원 구성
	3) UI 개선 방안 수립
	4) UI 개선 방안을 기반으로 수정 계획을 수립, 보고서 작성


9. ISO/IEC 9126의 소프트웨어 품질 특성 중 '기능성'
	기능성 (Functionality)
	소프트웨어가 특정 조건에서 사용될 때, 
	요구사항을 만족하는 기능을 제공하는 SW 제품의 능력을 이야기한다.
	부특성으로 성숙성, 결함 허용성, 회복성, 준수성 등이 있다.
