1. 주제별 체계
	- 주제나 화제에 따라 구분하는 방법


2. 사용자들이 이해하기 어려운 정보 레이블링
	- 그리드구조
	 바둑판식 구조, 정형화 쉽지 않음



3. 글로벌 네비게이션
-웹사이트 전체적으로 적용되는 구조, 네비바 형태로 구현
 메인 페이지의 상단에 위치, 서브 페이지에도 같은 자리에 위치
 사용자들이 서브 페이지로 이동해도 자신의 위치를 알기 쉽게 설계해야함


4. 구조도(사이트 맵)
-정보 공간의 형태를 개괄적으로 보여주며, 개발자와 사용자를 위한 요약 기능도 한다 
 페이지와 콘텐츠 간의 관계를 보여주므로 조직화, 내비게이션, 레이블링 시스템을 표현하는 용도로 사용

5. 와이어프레임의 정의
-대략적인 스크린 디자인, 
 페이지와 기능 간의 관계와 구조를 제안하고 시각화
 인터페이스 디자인에서 기본 안내서로 사용
 인터페이스의 기본 요소를 담은 레이아웃을 그림 등으로 표현한 것,
 기획서나 스토리보드 등의 UI 디자인을 의미
 가벼운 스케치와 인터랙티브 프로토타입 사이에 위치

6. 와이어프레임의 장점
-디자인을 좀 더 세밀하게 파악할 수 있게 한다.
 몇 장 묶으면 사용성 피드백을 얻을 정도의 완성도
 사용자에게 보여주고 어떻게 태스크를 완수할지 들여다 볼 수 있다.
 디자인과 기능에 대한 요구 사항을 명확히 전달할 수 있는 커뮤니케이션 도구
 디자인 명세 문서를 상당 부분 대체할 수 있다

7. 태스크 플로우
-사용자들이 웹사이트를 항해하는 중 취하는 경로나 과정을 보여준다. 
 사용자의 옵션이나 경로를 상세히 보여주기 위해 제작

8. 스케치하기
-전체적으로 문제를 조망하기에 유용한 방법.
 제품이나 기능을 시각화 할 때 가장 먼저 하는 일.
 동시에 요소 간의 연관성에 따라 어디에 배치할지 생각해야 한다. 
 빨리 그려보고 버릴 수 있는 스케치로 해보는게 적절.
